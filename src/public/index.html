Setup Guide

Form Fields
Client Name

Any label that helps you identify the client internally. Can be the company name or a shorthand like "acme-ops".

SmartLead API Key

Use this client’s own key — ideally a client-level key from SmartLead (agency mode), so it only ever sees that customer’s campaigns. The app checks every webhook: the campaign must belong to the key you paste here, or the event is ignored (nothing posted to Slack).

  1. Log in to SmartLead as the workspace that owns this customer’s campaigns
  2. Profile avatar (top right) → Settings → API Keys
  3. Create a key for that client (SmartLead’s docs: “client-level API keys” for agencies) and copy it here
Leave blank if this client doesn't run email campaigns.
HeyReach API Key

One HeyReach workspace = one API key. Generate the key inside that client’s HeyReach account so it only ever touches their campaigns and inboxes. We still verify each webhook’s campaign id against their key before posting to Slack.

  1. Log in to HeyReach as that client’s workspace (or their sub-account, however you separate clients)
  2. Settings → Integrations → API Keys → Generate New Key
Leave blank if this client doesn't run LinkedIn campaigns. Webhooks must include a campaignId (or campaign_id) so we can match the event to their campaigns.
Slack Bot Token
  1. Go to api.slack.com/apps and open your SalesGlider Slack app
  2. Click OAuth & Permissions in the left sidebar
  3. Scroll to Bot Token Scopes and add these (then Reinstall to Workspace so the token picks them up):
    • chat:write — required to post drafts and alerts
    • chat:write.public — post in channels the bot hasn’t joined yet (prefer inviting the bot anyway)
    • channels:history
    • im:write
    If Slack returns missing_scope, open the app’s OAuth & Permissions page — Slack often shows which scope is missing. After any scope change, copy the new xoxb- token into the client record in this dashboard.
  4. Under Interactivity & Shortcuts, set the Request URL to https://YOUR-RAILWAY-DOMAIN/slack/actions so Approve, Edit & send, and Reject work
  5. Scroll back up and copy the Bot User OAuth Token (starts with xoxb-)
Same token for all clients if they share the same Slack workspace. Different workspaces need different apps & tokens.
Slack Channel ID
  1. In Slack, create a dedicated channel for this client (e.g. #acme-replies)
  2. Invite the bot to the channel: type /invite @YourBotName in the channel
  3. Right-click the channel name in the sidebar → View channel details
  4. Scroll to the bottom — the Channel ID is listed there (starts with C)
Alternative: right-click channel → Copy link. The ID is the last segment of the URL.
Booking Link & Calendly token

Booking link — any scheduler (Calendly, Cal.com, SavvyCal, HubSpot meetings, etc.). The draft always includes this link when it’s time to book.

Calendly personal access token (optional) — only if the booking link is a calendly.com URL. With it, the app pulls real open times from Calendly so the AI can suggest two slots that actually exist. Without it (or for non-Calendly links), connect Google or Outlook on this client card so we can suggest times from calendar free/busy; otherwise the draft just points them to the link without inventing fake times.

Examples:

  • https://calendly.com/acme-ceo/30min
  • https://cal.com/acme/discovery-call
  • https://savvycal.com/acme/chat
Voice Prompt

2–4 sentences describing how Gemini should write on this client's behalf. Think tone, style, phrases to avoid, how to close.

Example:
"Direct, no-fluff tone. Speak like a fellow B2B operator. Never start with 'Great question' or 'Thanks for reaching out'. Keep replies to 2–3 sentences. Always end with a soft CTA for a 15-min call."
Meeting Booking Flow
How meetings work

When a prospect agrees to meet:

  1. AI drafts a short friendly reply
  2. If we have real availability (Calendly API and/or their connected calendar), the draft offers those times plus the booking link
  3. The message goes to Slack — someone taps Approve & send or Edit & send (or Reject)
  4. After approval, the reply goes out on SmartLead or HeyReach; optional calendar invite still depends on Google/Outlook being connected
Wrong webhook URL on someone else’s campaign? We skip it — nothing appears in the wrong client’s Slack.
Getting the client's booking link

Ask the client for the URL of their scheduling page. Any platform works:

  • Calendly — e.g. https://calendly.com/jane/30min
  • Cal.com — e.g. https://cal.com/acme/discovery
  • SavvyCal — e.g. https://savvycal.com/acme/chat
  • HubSpot — e.g. https://meetings.hubspot.com/jane

The scheduling platform handles calendar conflicts, Zoom/Meet/Teams links, confirmation emails, and reminders — we just need the URL.

After Creating the Client
Set up webhooks in SmartLead & HeyReach
  1. Click the client card in the dashboard to open its details
  2. Copy the SmartLead Webhook URL and paste it only on campaigns that belong to that same SmartLead API key’s account
  3. Copy the HeyReach Webhook URL and paste it only on campaigns in that HeyReach workspace
  4. Send a test reply — you should see a Slack card with Approve / Edit & send / Reject
Optional: set WEBHOOK_TEST_SECRET on the server and use POST /admin/test/slack-draft/<client-id> with header x-webhook-test-secret to post a fake card without a real prospect reply.
Railway (hosting)
Deploy checklist
  1. Connect this repo; Railway sets RAILWAY_PUBLIC_DOMAIN automatically
  2. Variables: DATABASE_URL, GEMINI_API_KEY, SLACK_SIGNING_SECRET, optional LEADMAGIC_API_KEY, optional WEBHOOK_TEST_SECRET, optional DEFAULT_BOOKING_TIMEZONE
  3. Run SQL: schema.sql on a fresh DB, or apply migrations/*.sql in order on an existing one (includes 004_calendly_pat.sql for the Calendly token column)
  4. Slack app → Interactivity URL = https://<your-domain>/slack/actions
